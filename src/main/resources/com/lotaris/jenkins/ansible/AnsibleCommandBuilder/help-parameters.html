The coma separated list of parameters to enrich the command.

Format supported:
paramName=paramValue,paramName=paramValue...

The parameter names will be replaced in the variables of the Ansible command throuhg a look of {{ paramName }}

Ansible configuration file:
project: projectName

properties:
  playbook: somePlaybook

inventory:
  - inventoryFile

vars:
  generalVariable: generalVariableValue

types:
  - name: vmType
		hosts:
			- host1
			- host2
    vars:
      specificVariable: specificVariableValue
			anotherVariableWithFiltering: {{ filteringToDo }}

And the following configuration: filteringToDo=someValue

In this case, the {{ filteringToDo }} will be replaced by someValue

There is a list of specific placeholder that will have a specific behavior. These variables are replaced in a specific way described in the following table.

Variable name Behavior
host Is the index in the list of hosts present on the VM type.
